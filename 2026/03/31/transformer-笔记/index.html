self-attention

什么是 self-attention

对于一句话的 词向量，经过 self-attention 之后，会变成维度相同，考虑过和其他位置的词的 词向量

self-attention 的具体流程

给定两个词向量 a1⃗\vec{a_1}a1​​ 和 a2⃗\vec{a_2}a2​​，如何计算这两个词向量的相似度呢

设 q1⃗=Wqa1⃗\vec{q_1} = W_q \vec{a_1}q1​​=Wq​a1​​，k2=Wka2⃗k_2 = W_k \vec{a_2}k2​=Wk​a2​​

向量 q⃗\vec{q}q​ 就表示 Query 查询

向量 k⃗\vec{k}k 表示 Key

attention score：a1,2=q1⃗⋅k2⃗a_{1 , 2} = \vec{q_1} \cdot \vec{k_2}a1,2​=q1​​⋅k2​​

然后对所有的 attention score 做 Soft-max：

然后 v1⃗=Wva1⃗\vec{v_{1}} = W_{v} \vec{a_1}v1​​=Wv​a1​​

向量 v⃗\vec{v}v 就表示内容资讯，**之所以需要向量 v⃗\vec{v}v 是因为需要在原信息中提取出有用信息。

每一个 v⃗\vec{v}v 乘上对应的 attention score，attention score 越大，这个 v⃗\vec{v}v 越容易被抽取出来，最后得到 b1⃗\vec{b_1}b1​​

self-attention 的矩阵表示和理解

一个 a⃗\vec{a}a 向量，左乘一个矩阵 WqW_{q}Wq​，得到一个 q⃗\vec{q}q​ 向量，那把 a⃗\vec{a}a 向量横着堆在一起，就有了 q⃗\vec{q}q​ 向量堆在一起（堆在一起，也就形成了 QQQ 矩阵）

把 k⃗\vec{k}k 向量转置（从竖着，变成横着），然后乘向量 q⃗\vec{q}q​，就得到了一个 attention score

对于 q1⃗\vec{q_1}q1​​，把 k1⃗\vec{k_{1}}k1​​ 竖着堆积起来，就得到了一列 attention score

然后再把向量 q⃗\vec{q}q​ 横着堆积起来，就得到了 attention score 的矩阵

随后做 softmax 的时候，就要对于每一列做 softmax 了

然后发现对于其中的一列元素，需要和 v1⃗\vec{v_1}v1​​、v2⃗\vec{v_2}v2​​、v3⃗\vec{v_3}v3​​ … 对应相乘，于是把 v1⃗\vec{v_1}v1​​、v2⃗\vec{v_2}v2​​、v3⃗\vec{v_3}v3​​ 横着堆起来，和刚刚 softmax 过的 attention score 矩阵相乘即可

整个过程，只有 WqW_{q}Wq​、WkW_{k}Wk​、WvW_{v}Wv​ 三个矩阵的参数需要训练

时间复杂度

Attention 矩阵（n×nn \times nn×n）左乘 VVV 矩阵的时间复杂度：(512,n)×(n,n)(512 , n) \times (n , n)(512,n)×(n,n)，时间复杂度 O(n2×d)O(n^2 \times d)O(n2×d)

由于一般 nnn 量级高于 ddd，所以复杂度是 O(n2d)O(n^2 d)O(n2d)

为什么要除以 dk\sqrt{d_k}dk​​

Var\text{Var}Var 表示方差

Var(qi⃗⋅kj⃗)=∑Var(qiki)\text{Var}(\vec{q_i} \cdot \vec{k_j}) = \sum \text{Var}(q_i k_i)Var(qi​​⋅kj​​)=∑Var(qi​ki​)，一共 dkd_kdk​ 项。因为提前通过特定的随机化的，保证了 Var(qi)\text{Var}(q_i)Var(qi​) 和 Var(ki)\text{Var}(k_i)Var(ki​) 都是 111，所以最后 Var(qi⃗⋅kj⃗)=dk\text{Var}(\vec{q_i} \cdot \vec{k_j}) = d_kVar(qi​​⋅kj​​)=dk​

根据Var\text{Var}Var Var\text{Var}Var 的一条性质：

Var(c⋅X)=c2Var(X)\text{Var}(c \cdot X) = c^2 \text{Var}(X)Var(c⋅X)=c2Var(X)

所以 Var(q⃗⋅k⃗dk)=dk(dk)2=1\text{Var} ( \dfrac{\vec{q} \cdot \vec{k}} {\sqrt{d_k}} ) = \dfrac{d_k}{(\sqrt{d_k})^2} = 1Var(dk​​q​⋅k​)=(dk​​)2dk​​=1

假设两个 token 的点积得分分别是 [2, 3]

softmax([2, 3]) = [0.27, 0.73] ← 还算平滑

softmax([20, 30]) = [0.00005, 0.99995] ← 几乎变成 one-hot！

softmax 进入饱和区之后：

  • 梯度 → 接近 0（梯度消失）

  • 注意力退化成"只看一个 token"，失去软注意力的意义

方差被归一化回 1，无论 dkd_kdk​ 多大，点积的量级都保持稳定，softmax 的输入不会爆炸。

Q K V 是什么，为什么要分成 3 个

Q 表示 Query 查询

K 表示 key

V 表示内容资讯

  • 为什么是 KTQK^TQKTQ 不是 QTQQ^T QQTQ：因为注意力不能对称，我关注小明，和小明关注我肯定是不能一样的

  • 为什么需要 VVV，因为 VVV 可以在原来的词向量中学习到有用的知识

Attention 矩阵的物理含义

值越大，说明这个 token 在当前语境 下对目标 token 越重要

为什么用点积求相似度

因为点积求相似度，可以进一步让计算转化成 矩阵乘法，能够用 GPU 并行计算

加性（Additive）相似度：理论表达能力更强，但是太慢，无法转化成矩阵乘法

余弦相似度：仅考虑方向，对长度（强度）不敏感，固定在 [-1 , 1]。计算的时候还要算模长、除法，降低 吞吐量

Multi-Head 的作用是什么，为什么不用一个大的 head

让模型能够学习到不同层面的语义，信息更丰富：学习语法，学习语义上的相似度等等

Multi-Head 不是参数更多，而是把同样的参数拆开，强迫模型从多个角度理解语言，比一个大 head 表达能力更强。

为什么需要 WOW_OWO​？拼接不够吗？

因为要深度融合多个头的资讯，并且 WOW_OWO​ 是可以学习的参数，可以更好的提取有用的组合特征

self-attention VS RNN

RNN 处理顺序是 串行 的，一个 token 一个 token 按顺序处理

Self-attention 处理顺序是 并行的，所有 token 同时计算

RNN 没有办法并行计算，训练速度慢

RNN 随着序列的加长，容易出现 梯度消失，效果差

RNN 的串行结构天然携带顺序信息，self-attention 并行处理所有 token，对顺序“视而不见”，所以需要 位置编码 来补充这个信息

杂项

梯度消失 g<1g < 1g<1：

假设每层梯度 g = 0.5，网络有 10 层：

第10层（输出层附近）：梯度 = 0.5¹ = 0.5 ← 正常

第 5层： 梯度 = 0.5⁵ = 0.03 ← 变小了

第 1层（输入层附近）：梯度 = 0.5¹⁰ = 0.001 ← 几乎为 0

结果：靠近输入层的 参数几乎不更新，相当于 没有在学习。

梯度爆炸 g>1g > 1g>1：

假设每层梯度 g = 2，网络有 10 层：

第 10 层：梯度 = 2¹ = 2

第 5 层：梯度 = 2⁵ = 32

第 1 层：梯度 = 2¹⁰ = 1024 ← 爆炸！

结果：参数更新 步子迈得太大，训练震荡、发散，Loss 不降反升。

矩阵乘法的时间复杂度

对于一个 d×dd \times dd×d 的矩阵 AAA，和一个 d×nd \times nd×n 的矩阵 BBB 相乘，复杂度是 AAA 的行大小和列大小相乘，然后乘上 BBB 的列的个数，复杂度 O(n×d2)O(n\times d^2)O(n×d2)

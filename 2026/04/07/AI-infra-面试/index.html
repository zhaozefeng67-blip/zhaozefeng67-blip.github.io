AI Infra 面试

Cache

存储系统存在的问题：

  • 双端口 RAM、多模块存储器提高存储器的工作速度

  • 优化后速度与 CPU 差距依然很大

加入微信视频聊天一直被频繁的访问，那直接把在内存 (4GB) 中和视频聊天的代码复制一份到 Cache（4MB），然后 CPU 直接在 Cahce 中取指令、读数据，速度很快。

实际上，Cache 被集成在 CPU 内部。Cache 用 SRAM（SRAM 比 DRAM 要快的多，但成本也会高） 实现，速度快，成本高。

SRAM 的集成度比较低，这就意味着如果我们想把 Cache 塞到一个很小的手机里面，由于集成度比较低，并且芯片也不能做的特别大，注定了 Cache 存储空间不可能做的特别大

i5 系列 CPU 的 cache 通常也就是 12 MB

看起来数据是二维数组，但是放到内存里，是一行的。

空间局部性：在最近的未来很有可能要用到的信息（指令和数据），很有可能与现在正在使用的信息在存储空间上是邻近的。

时间局部性：在最近未来很有可能要用到的信息，很可能是现在正在使用的信息

基于局部性原理，不难想到，把 CPU 目前访问的地址“周围”的部分数据放到 Cache 中

如果把程序改成：一列一列访问，那么在内存读取信息的时候，就得跳着读取，空间局部性要更差。

KV cache

每一次生成新的 token 时，都要把新加入的词向量，以及之前的词向量送入 self-attention，然后计算 Q K V，但是没有必要把之前的词向量再计算一次 k 向量和 v 向量，所以九把之前的 k 向量和 v 向量都存起来了
